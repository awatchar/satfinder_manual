SatFinder x TinyGS

คู่มือการใช้งาน TinyGS Dashboard สำหรับครูผู้สอน

เอกสารประกอบการอบรมโครงการส่งเสริมการเรียนรู้ทางด้านโทรคมนาคมในโรงเรียนทั่วประเทศ (SatFinder) โดยคณะวิศวกรรมศาสตร์ มหาวิทยาลัยธรรมศาสตร์ และสถาบันวิจัยและให้คำปรึกษาแห่งมหาวิทยาลัยธรรมศาสตร์ สนับสนุนโดย กสทช. เพื่อประโยชน์สาธารณะ

วัตถุประสงค์คู่มือ

  • ช่วยครูเข้าใจ TinyGS Dashboard อย่างเป็นลำดับ
  • เตรียมเนื้อหาก่อนนำไปสอนขยายผลกับนักเรียน
  • ใช้ข้อมูลจริงเพื่อฝึกการคิดเชิงวิศวกรรมและการสื่อสาร

1) เนื้อหาพื้นฐานที่ครูควรรู้ก่อนใช้งาน TinyGS

1.1 TinyGS คืออะไร และใช้ทำอะไรในงานการเรียนการสอน

TinyGS เป็นแพลตฟอร์มโอเพ่นซอร์สและชุมชนผู้ใช้งานทั่วโลกที่สนับสนุนการรับสัญญาณดาวเทียมขนาดเล็ก (โดยเฉพาะกลุ่ม CubeSat และดาวเทียมการศึกษา) ผ่านเครือข่ายสถานีรับสัญญาณภาคพื้น (Ground Station) แบบต้นทุนต่ำ โดยมี Dashboard เป็นศูนย์กลางสำหรับการสังเกตการณ์และวิเคราะห์ข้อมูล เช่น packets, telemetry, satellite pass และข้อมูลการรับสัญญาณจากสถานีต่าง ๆ

ในมิติการเรียนการสอน TinyGS Dashboard เปรียบเสมือน “ห้องทดลองเสมือน” ที่ทำให้ครูและนักเรียนเรียนรู้ แนวคิดด้านโทรคมนาคมและอวกาศด้วยข้อมูลจริง (real-world data) ได้ตั้งแต่เริ่มต้น แม้ยังไม่มีอุปกรณ์ภาคสนามครบชุด

ระดับสังเกตการณ์ (Observation)

  • เรียนรู้ว่าดาวเทียมส่งข้อมูลอย่างไร และข้อมูลแสดงบน Dashboard แบบไหน
  • เหมาะกับพื้นฐานคลื่นวิทยุ, ข้อมูลดิจิทัล, telemetry และเครือข่ายสถานีภาคพื้น

ระดับวิเคราะห์ข้อมูล (Data Interpretation)

  • วิเคราะห์ความถี่การรับแพ็กเก็ตและความสัมพันธ์กับช่วง pass
  • เปรียบเทียบผลลัพธ์ระหว่างสถานี และฝึกมองคุณภาพสัญญาณ/packet loss

ระดับออกแบบการทดลอง (Experiment Design)

  • ตั้งสมมติฐาน เช่น max elevation สูงขึ้นจะรับได้มากขึ้นหรือไม่
  • เชื่อมโยง STEM: สถิติพื้นฐาน, การคิดเชิงระบบ และการสื่อสารเชิงวิศวกรรม

ข้อดีเชิงการสอน: Dashboard ช่วยลดอุปสรรคการเริ่มต้น เพราะผู้เรียนเข้าถึงข้อมูลเพื่อฝึกตีความและตั้งคำถามทางวิศวกรรมได้ ก่อน เมื่อโรงเรียนพร้อมจึงค่อยต่อยอดสู่การติดตั้งสถานี TinyGS จริงและทำโครงงานแบบ project-based learning

1.2 ขั้นตอนเข้าหน้าเว็บไซต์และลงชื่อเข้าใช้

หมายเหตุ: หากจุดประสงค์คือ “สังเกตการณ์/วิเคราะห์ข้อมูล” โดยทั่วไปสามารถทำได้แม้ไม่ลงชื่อเข้าใช้ (ขึ้นกับนโยบายการเข้าถึงของ Dashboard ในเวลานั้น)

กรณี A: ดูและวิเคราะห์ข้อมูล (ไม่ต้องล็อกอิน)

  1. เปิดเว็บเบราว์เซอร์ (แนะนำ Chrome/Edge รุ่นปัจจุบัน)
  2. เข้าสู่หน้า TinyGS Dashboard จากลิงก์ในคู่มือโครงการ SatFinder
  3. ตรวจสอบองค์ประกอบหลัก เช่น แผนที่, แถบเมนู, รายการข้อมูล
  4. เริ่มเลือกดาวเทียมหรือมุมมองข้อมูลที่ต้องการ

ผลลัพธ์ที่คาดหวัง: ครูควรเห็นข้อมูลแบบสาธารณะอย่างน้อยระดับรายการดาวเทียม/สถิติพื้นฐาน

กรณี B: ลงชื่อเข้าใช้ (เพื่อความสามารถเพิ่มเติม)

  1. เข้าหน้า TinyGS Dashboard
  2. มองหาปุ่ม/เมนู Sign in, Login หรือ Account (มักอยู่มุมขวาบน)
  3. เลือกวิธีลงชื่อเข้าใช้ตามที่ระบบรองรับ
  4. ตรวจสอบชื่อผู้ใช้หรือเมนูที่เพิ่มขึ้นหลังล็อกอินสำเร็จ

ใช้เมื่อต้องการตั้งค่าเฉพาะบัญชี จัดการสถานี หรือฟังก์ชันเฉพาะโรงเรียน

ข้อควรระวัง (Caution)

  • ใช้รหัสผ่านไม่ซ้ำและไม่แชร์ในที่สาธารณะ
  • หลีกเลี่ยงการบันทึกรหัสผ่านบนเครื่องสาธารณะ
  • แยกบัญชีสาธิตในห้องเรียนออกจากบัญชีที่ใช้ตั้งค่าระบบจริง

1.3 คำอธิบายเมนูหลักที่ควรรู้ก่อนเริ่มใช้งานจริง

ชื่อเมนูใน Dashboard อาจต่างกันเล็กน้อยตามรุ่น/การปรับ UI แต่แนวคิดหลักใกล้เคียงกัน ด้านล่างคือกลุ่มเมนูสำคัญที่ครูควรเข้าใจเพื่อใช้สอนได้อย่างมั่นใจ

1) Satellite / Satellites

ค้นหาและเลือกดาวเทียมที่ต้องการติดตาม ใช้ตั้งโจทย์ “เลือกดาวเทียม 1 ดวง” เป็นแกนกิจกรรมทั้งคาบ

2) Pass / Passes / Next Pass

ดูช่วงเวลาที่รับสัญญาณได้ พร้อมตัวชี้วัดเวลาเริ่ม-สิ้นสุดและมุมเงยสูงสุด เพื่อสอนเรื่องหน้าต่างเวลาสื่อสาร

3) Telemetry / Packets / Frames

หัวใจของการวิเคราะห์ข้อมูลจริง เช่น timestamp, source, payload ฝึกแยกความต่างระหว่าง “สัญญาณ” และ “ข้อมูล”

4) Map / Stations / Nodes

ดูการกระจายของสถานีภาคพื้นและสถานะเครือข่าย ใช้สอน distributed network ในระบบอวกาศ

5) Statistics / Activity

ใช้ดูจำนวนแพ็กเก็ตและแนวโน้มตามเวลา เพื่อต่อยอดกิจกรรมสถิติพื้นฐานและการทดสอบสมมติฐาน

6) Search / Filter และ 7) Account / Settings

ช่วยกรองข้อมูลไม่ให้ล้น และใช้การตั้งค่าเฉพาะเมื่อจำเป็น โดยกำหนดขอบเขตการปรับแต่งให้ชัดเจน

2) โครงร่างหัวข้อสอน + จุดวางภาพหน้าจอ TinyGS

รองรับการเติมภาพ Print Screen ทีละส่วนเมื่อได้รับไฟล์จากทีมงาน

  • โครงสร้างหน้าจอหลัก, แถบเมนู, และตำแหน่งข้อมูลสำคัญ
  • แนวทางแนะนำผู้เรียนก่อนเริ่มกิจกรรมจริง
พื้นที่วางภาพ: หน้าแรกของ TinyGS Dashboard

  • คลิกเมนู Hamburger Icon (มุมซ้ายบน) แล้วเลือกเมนู Station
  • พิมพ์ในช่อง Find Station ตามรูปแบบ SatFinder_จังหวัด_ชื่อย่อโรงเรียน
  • เลือก Station ของโรงเรียนจากผลลัพธ์เพื่อดูสถานะและข้อมูลการรับสัญญาณ

ตัวอย่างการค้นหา: SatFinder_ปทุมธานี_TUS

พื้นที่วางภาพ: เมนู Station + ช่อง Find Station (ตามภาพตัวอย่างที่ได้รับ)

  • อ่านช่วงเวลาที่ดาวเทียมผ่านและโอกาสรับสัญญาณ
  • ตีความข้อมูล timestamp, packet และค่าเชิงคุณภาพสัญญาณ
  • ตั้งคำถามให้นักเรียนวิเคราะห์จากข้อมูลจริง
พื้นที่วางภาพ: หน้า Passes / Telemetry / Packets

  • กิจกรรม 15–30 นาทีแบบพร้อมใช้ในห้องเรียน
  • ใบงานฝึก data literacy และการสื่อสารผลวิเคราะห์
  • แนวทางต่อยอดเป็นโครงงาน STEM ของโรงเรียน
พื้นที่วางภาพ: ตัวอย่างใบงาน/กิจกรรมจากหน้าจอจริง
